# ระบบบันทึกปริมาณการใช้น้ำประปาและน้ำบาดาล
## เวอร์ชัน HTML + Google Sheets (ไม่ต้องมี Server / ไม่ต้องใช้ PHP)

ระบบเดียวกับเวอร์ชัน PHP ทุกฟีเจอร์ (บันทึกค่ามาตร, คำนวณ usage อัตโนมัติ,
ตรวจสอบถังเก็บน้ำ, แนบรูปถ่าย, แดชบอร์ด+กราฟ, ประวัติข้อมูล) แต่เปลี่ยนมาใช้:

- **Google Sheets** เป็นฐานข้อมูล (แทน MySQL)
- **Google Apps Script** เป็น backend/API (แทน PHP)
- **ไฟล์ HTML+JavaScript ล้วน** เป็นหน้าเว็บ (วางไว้ที่ไหนก็ได้ ไม่ต้องมี server)
- **ระบบ Login** ตรวจสอบสิทธิ์จริงฝั่ง Server (Apps Script) ทุกครั้งที่มีการอ่าน/เขียนข้อมูล
  ไม่ใช่แค่ซ่อนหน้าเว็บไว้เฉยๆ — ต่อให้ใครเห็นซอร์สโค้ด HTML ก็ยังต้องมี username/password
  ที่ถูกต้องจึงจะดึงหรือแก้ไขข้อมูลได้

## ขั้นตอนติดตั้ง (ทำครั้งเดียว ใช้เวลาประมาณ 10-15 นาที)

### 1. สร้าง Google Sheet
1. ไปที่ [sheets.google.com](https://sheets.google.com) → สร้างสเปรดชีตใหม่
2. ตั้งชื่อ เช่น "ฐานข้อมูลระบบน้ำโรงพยาบาล"
3. ไม่ต้องสร้างชีทย่อยเอง — สคริปต์จะสร้าง Records / Tanks / TankChecks / Recorders /
   Users / Sessions ให้อัตโนมัติ

### 2. เพิ่ม Apps Script
1. ในสเปรดชีต ไปที่เมนู **ส่วนขยาย (Extensions) → Apps Script**
2. ลบโค้ดเดิมทั้งหมดในไฟล์ `Code.gs` แล้ววางโค้ดจากไฟล์ `gas/Code.gs` ที่แนบมาให้แทน
3. กด **บันทึก** (ไอคอนแผ่นดิสก์)
4. ที่แถบด้านบน เลือกฟังก์ชัน `setupSampleData` แล้วกด **▶ Run** (รัน) เพื่อสร้างถังเก็บน้ำ,
   ข้อมูลตัวอย่าง, และ**บัญชีผู้ใช้เริ่มต้น**ให้อัตโนมัติ (ครั้งแรกจะมีหน้าต่างขอสิทธิ์
   ให้กด "อนุญาต")

   บัญชีเริ่มต้นที่สร้างให้คือ:
   - **Username:** `admin`
   - **Password:** `admin1234`

   ⚠️ **สำคัญมาก:** เปลี่ยนรหัสผ่านนี้ทันทีหลังติดตั้งเสร็จ (ดูขั้นตอนที่ 6)

### 3. Deploy เป็น Web App
1. มุมขวาบนของ Apps Script กด **Deploy → New deployment**
2. เลือกประเภท (Select type) เป็น **Web app**
3. ตั้งค่า:
   - Execute as: **Me (อีเมลของคุณ)**
   - Who has access: **Anyone** (สำคัญ — ต้องเป็น Anyone เพื่อให้หน้าเว็บเรียกใช้ได้
     การจำกัดสิทธิ์ที่แท้จริงทำผ่านระบบ Login ในแอป ไม่ใช่ผ่านการตั้งค่านี้)
4. กด **Deploy** → ระบบจะขอ authorize อีกครั้ง กด "Advanced" → "Go to project (unsafe)" → Allow
   (ข้อความเตือนนี้เป็นเรื่องปกติ เพราะเป็นสคริปต์ที่คุณเขียน/วางเอง)
5. คัดลอก **Web app URL** ที่ได้ (หน้าตาแบบ `https://script.google.com/macros/s/AKfycb.../exec`)

### 4. เชื่อมหน้าเว็บเข้ากับ Apps Script
1. เปิดไฟล์ `web/index.html` ด้วยโปรแกรมแก้ไขข้อความ (Notepad, VS Code ฯลฯ)
2. หาบรรทัด:
   ```js
   const APPS_SCRIPT_URL = 'https://script.google.com/macros/s/XXXXXXXXXXXXXXXXXXXXXXXX/exec';
   ```
3. แทนที่ด้วย URL ที่คัดลอกมาจากขั้นตอนที่ 3
4. บันทึกไฟล์

### 5. เปิดใช้งาน
เลือกวิธีใดวิธีหนึ่ง:

- **ง่ายที่สุด**: ดับเบิลคลิกไฟล์ `index.html` เปิดด้วยเบราว์เซอร์ได้ทันที (ใช้งานคนเดียวในเครื่อง)
- **ใช้งานร่วมกันหลายคน/หลายเครื่อง**: อัปโหลดไฟล์ `index.html` ขึ้นที่ใดที่หนึ่งที่มี URL ให้เข้าถึงได้ เช่น
  - **GitHub Pages** (ฟรี, ต้องเป็น repo แบบ Public — ดูหัวข้อ "เรื่อง Public/Private repo" ด้านล่าง)
  - **Google Sites** — แทรกเป็น embed หรือแนบลิงก์
  - **Netlify / Vercel** (ฟรี, รองรับ private repo + site เป็น public URL) — ลาก-วางไฟล์เพื่อ deploy
  - หรือเว็บโฮสติ้งใด ๆ ของโรงพยาบาลที่มีอยู่แล้ว (ไม่จำเป็นต้องรองรับ PHP อีกต่อไป
    เพราะไฟล์นี้เป็น HTML/JS ล้วน)

จากนั้นเข้าใช้งานผ่านมือถือ/คอมพิวเตอร์ได้จาก URL นั้นทันที — จะเจอหน้า **เข้าสู่ระบบ**
ก่อนเสมอ กรอก username/password ที่ตั้งไว้เพื่อเข้าใช้งาน

### 6. เปลี่ยนรหัสผ่าน / เพิ่มผู้ใช้ / ปิดสิทธิ์ผู้ใช้
กลับไปที่ **Apps Script editor** แล้วรันฟังก์ชันเหล่านี้ผ่านเมนู Run (เลือกฟังก์ชันที่ต้องการ
จาก dropdown ด้านบน แล้วกด ▶ Run):

- **เปลี่ยนรหัสผ่าน admin**: แก้บรรทัดใน editor ชั่วคราวเป็น
  `createUser('admin', 'รหัสผ่านใหม่ที่ปลอดภัย', 'ผู้ดูแลระบบ')` แล้วรันครั้งเดียว
- **เพิ่มผู้ใช้ใหม่**: `createUser('username', 'password', 'ชื่อที่แสดงผล')`
- **ปิดสิทธิ์ผู้ใช้** (ไม่ลบข้อมูล แค่ระงับการเข้าสู่ระบบ): `disableUser('username')`

หรือจะเปิดชีท **Users** แล้วดูรายชื่อผู้ใช้ทั้งหมดตรงๆ ก็ได้ (แต่ห้ามแก้คอลัมน์
PasswordHash เอง เพราะเป็นค่าที่เข้ารหัสไว้แล้ว ต้องใช้ฟังก์ชัน `createUser` เท่านั้น)

## เรื่อง Public / Private repo บน GitHub

GitHub Pages แบบฟรี (personal free plan) **ใช้ได้เฉพาะ repository ที่เป็น Public เท่านั้น**
ถ้าเปลี่ยน repo เป็น Private เว็บไซต์จะไม่แสดงผล (ต้องอัปเกรดเป็น GitHub Pro ขึ้นไปถึงจะ
ใช้ Pages กับ private repo ได้)

**คำแนะนำ**: ต่อให้ repo เป็น Public ก็ไม่ได้แปลว่าข้อมูลไม่ปลอดภัย เพราะ:
- ไฟล์ที่อยู่ใน repo คือ `index.html` เพียงไฟล์เดียว ไม่มีข้อมูลผู้ป่วย/มาตรน้ำอยู่ในนั้นเลย
  (ข้อมูลทั้งหมดอยู่ใน Google Sheet ที่คุณเป็นเจ้าของ ไม่ได้อยู่ใน repo)
- ต่อให้เห็น URL ของ Apps Script ในซอร์สโค้ด ก็ยัง**เข้าสู่ระบบด้วย username/password
  ไม่ได้ถ้าไม่รู้รหัสผ่านจริง** เพราะ Apps Script ตรวจสอบสิทธิ์ฝั่ง Server ทุกครั้ง
  (ระบบ Login ที่เพิ่มเข้ามาป้องกันจุดนี้ไว้แล้ว)

ถ้ายังต้องการ repo เป็น Private จริงๆ (เช่น นโยบายของโรงพยาบาล) แนะนำให้ย้ายไป deploy ที่
**Netlify** หรือ **Vercel** แทน (ฟรี, เก็บซอร์สโค้ดใน private repo ได้ แต่เว็บที่ deploy
ออกมายังเป็น public URL เหมือนเดิม) หรือใช้แผน **GitHub Pro** (ประมาณ 4 ดอลลาร์/เดือน)
ซึ่งเปิด Pages จาก private repo ได้โดยตรง

## โครงสร้างข้อมูลใน Google Sheet (สร้างอัตโนมัติ)

| ชีท | คอลัมน์ |
|---|---|
| **Records** | ID, Date, TapMeter, TapUsage, GroundMeter, GroundUsage, Recorder, PhotoURL, Note, CreatedAt |
| **Tanks** | ID, TankName, Order, Active |
| **TankChecks** | ID, RecordID, TankID, Checked, Note |
| **Recorders** | ID, Name |
| **Users** | ID, Username, PasswordHash, DisplayName, Active |
| **Sessions** | Token, Username, ExpiresAt (หมดอายุอัตโนมัติหลัง 12 ชั่วโมง) |

สามารถเปิดดู/แก้ไขข้อมูลตรงในสเปรดชีตได้เลยเหมือน Excel ปกติ — เหมาะกับผู้ดูแลระบบ
ที่อยากตรวจสอบหรือ export ข้อมูลไปทำรายงานเพิ่มเติม

รูปถ่ายมาตรน้ำจะถูกอัปโหลดไปเก็บไว้ในโฟลเดอร์ Google Drive ชื่อ **WaterSystemPhotos**
โดยอัตโนมัติ (สร้างให้เองครั้งแรกที่มีการอัพโหลดรูป)

## ข้อดี / ข้อจำกัด เทียบกับเวอร์ชัน PHP

**ข้อดี**
- ไม่ต้องมีเซิร์ฟเวอร์ ไม่ต้องมีทีม IT ดูแล MySQL
- ดูแก้ไขข้อมูลตรงในสเปรดชีตได้เหมือน Excel
- ฟรี 100% ใช้โควตาของ Google
- มีระบบ Login แยกผู้ใช้แต่ละคนได้ (เห็นชื่อผู้เข้าสู่ระบบ, ปิดสิทธิ์เฉพาะคนได้)

**ข้อจำกัด**
- ความเร็วขึ้นกับ Google Apps Script (ช้ากว่า MySQL เล็กน้อยเมื่อข้อมูลเยอะมาก ๆ หลักหมื่นแถวขึ้นไป)
- ต้องมีบัญชี Google ในการสร้าง/ดูแลสเปรดชีตและสคริปต์
- ระบบ Login นี้เหมาะกับการใช้งานภายในหน่วยงาน (ผู้ใช้ไม่กี่สิบคน) ยังไม่มีฟีเจอร์ระดับองค์กร
  เช่น สิทธิ์แยกตามบทบาท (role-based access), บังคับเปลี่ยนรหัสผ่าน, หรือ 2FA

## แก้ปัญหาที่พบบ่อย

- **กดบันทึกแล้วไม่มีอะไรเกิดขึ้น / ค้างที่ "กำลังบันทึก"** → ตรวจสอบว่าตั้งค่า
  `APPS_SCRIPT_URL` ถูกต้อง และตอน Deploy เลือก "Who has access: Anyone" แล้ว
- **ขึ้น error "Authorization required"** → ต้อง Deploy ใหม่และกด Allow สิทธิ์ให้ครบ
- **แก้โค้ดใน Apps Script แล้วเว็บไม่เปลี่ยน** → ต้องกด **Deploy → Manage deployments →
  แก้ไข (ไอคอนดินสอ) → New version** ทุกครั้งที่แก้โค้ด ไม่เช่นนั้น URL เดิมจะยังใช้โค้ดเวอร์ชันเก่า
- **เข้าสู่ระบบไม่ได้ / ขึ้น "ชื่อผู้ใช้หรือรหัสผ่านไม่ถูกต้อง"** → ตรวจสอบว่ารัน
  `setupSampleData` แล้วอย่างน้อยหนึ่งครั้ง (ดูขั้นตอนที่ 2) หรือเปิดชีท Users
  เพื่อตรวจสอบว่ามีรายชื่อผู้ใช้และคอลัมน์ Active เป็น 1 หรือไม่
- **ถูกเด้งออกจากระบบบ่อย** → ค่าเริ่มต้น session จะหมดอายุทุก 12 ชั่วโมง แก้ได้ที่ตัวแปร
  `SESSION_HOURS` ใน `Code.gs`
